Yalla

Plateforme web de planification de voyages permettant de créer des itinéraires collaboratifs, organiser des activités par jour et visualiser les trajets sur carte.

Stack — FastAPI · Ruby on Rails · PostgreSQL · jQuery · Bootstrap · Google Maps

← Retour aux projets

Overview

Yalla est une application de planification de voyages permettant aux utilisateurs de créer des itinéraires personnalisés, collaborer avec des amis et organiser leurs activités sous forme de colonnes par jour (UX inspirée de Trello). Le projet combine un backend API développé en FastAPI et une application web en Ruby on Rails.

What I did

1. Développement du backend API (FastAPI)

J’ai développé une API REST en FastAPI permettant de gérer les entités principales : utilisateurs, voyages, activités, invitations et votes. L’API expose des endpoints sécurisés et documentés via Swagger, facilitant les tests et l’intégration frontend.

2. Interface de planification drag & drop

L’interface principale permet d’organiser les activités par jour grâce à un système de drag & drop dynamique (jQuery UI). Chaque modification est persistée en base et reflétée en temps réel.

3. Visualisation cartographique

Intégration de Google Maps pour afficher les activités et optimiser la compréhension géographique du voyage. Les marqueurs sont générés dynamiquement via endpoint JSON.

4. Authentification & collaboration

Mise en place d’une authentification via Devise et OmniAuth (Facebook). Gestion des participants, invitations par email et votes sur les activités, permettant une vraie planification collaborative.

5. Architecture et évolutions

Le projet a évolué progressivement vers une architecture plus structurée, avec séparation claire entre API backend (FastAPI) et application web (Rails). Certaines itérations récentes ont été accélérées grâce à des outils d’assistance au code, tout en conservant l’architecture et la logique métier développées initialement.

Tech & tools

FastAPI pour le backend REST, Ruby on Rails 7 pour l’application web, Bootstrap pour le design responsive, jQuery pour les interactions dynamiques (drag & drop), PostgreSQL comme base relationnelle, Devise + OmniAuth pour l’authentification, Google Maps API pour la visualisation cartographique.

Évolutions prévues

Le projet est en cours d’évolution avec l’intégration future de fonctionnalités IA : recommandations d’activités optimisées selon les trajets, génération automatique de planning intelligent, suggestions contextuelles entre deux points du voyage.
